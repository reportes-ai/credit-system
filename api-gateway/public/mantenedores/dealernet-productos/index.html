Productos DealerNet — Central de Información

Activa los productos que el negocio consulta por RUT vía Web Service. Solo los activos se piden en cada consulta (cuidan el saldo).

Las credenciales del Web Service van como variables de entorno en Render (DEALERNET_USER / DEALERNET_PASS), nunca en el código. Mientras no estén cargadas, la consulta avisará "credenciales no configuradas".

Qué afecta este mantenedor

  • Productos activos → definen qué pide cada Consulta DealerNet por RUT. Menos productos activos = menos saldo por consulta.
  • Código / Nombre → identifican cada producto del catálogo DealerNet (rango 3401–3450 y códigos propios como 16 Comportamiento Vigente o 2101 Boletín Deudores de Pensión). Agrega aquí los códigos nuevos que te confirme DealerNet.
  • Vigencia de informes (bloqueo / vigencia) → controlan la reutilización del repositorio en Informes DealerNet: bajo el bloqueo no deja consultar (usa histórico), entre bloqueo y vigencia pregunta, sobre la vigencia consulta directo.
  • Credenciales → en variables de entorno de Render; afectan a toda consulta al Web Service.
  • El módulo Informes DealerNet (consulta en vivo + repositorio compartido) reemplaza la carga manual de PDF de Informes Comerciales Dealernet.
