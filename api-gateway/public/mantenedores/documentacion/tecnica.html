Documentación Técnica

Arquitectura, tablas, motores e integraciones del Business Suite. Referencia para desarrollo y para el Administrador técnico.

Stack y arquitectura

BackendExpress.js + Node.js — microservicios independientes en /services/*/src/{controllers,routes}, montados por el api-gateway.
Base de datosTiDB Cloud (MySQL-compatible). Pool único compartido: shared/config/database.js. Timezone del pool alineado a Chile (gotcha mysql2: debe coincidir con SET time_zone o las horas se corren 4h).
FrontendHTML/CSS/JS vanilla en api-gateway/public/. PWA instalable (manifest + sw sin caché). App dedicada AF WhatsApp con manifest propio (scope /whatsapp/).
DeployGitHub main → Render auto-deploy (~2-3 min). APP_VERSION en public/js/app-version.js confirma el deploy en el badge de todas las páginas.
AuthJWT (shared/middleware/auth.js). Tokens separados: interno (usuarios), tipo=dealer (portal dealer), tipo=cliente (portal cliente). Permisos por requireFunc('codigo') contra la matriz en BD (Admin bypass, caché 60s).
MigracionesIIFE (async () => {...})() al inicio de cada controller: CREATE TABLE IF NOT EXISTS + ALTER ADD COLUMN IF NOT EXISTS + seeds INSERT IGNORE. Pendiente: runner central (capataz) que las serialice.

Máximas de diseño (no negociables)

1. Un solo motor por cálculo: cada magnitud se calcula en UNA función compartida que usan todos los consumidores. 2. Una sola fuente de datos: cada dato vive en una tabla autoritativa; el resto la referencia por JOIN/FK. Snapshot deliberado (lo congelado al otorgar) ≠ duplicado. 3. Anti-hardcode: módulos y cards salen de las tablas modulos/funcionalidades, nunca de listas JS.

estados_credito / estados_transicionMáquina de estados paramétrica por ámbito.

Motores de cálculo (un solo hogar cada uno)

MotorQué calcula y quién lo usa
rentabilidad-core.jsCuota francesa, ingresos por tasa y seguros, rentabilidad. Isomorfo (module.exports + window.AF_RENT). Lo usan cartas, simulador, dashboard (recalcular-mes) y el bot.
shared/cotizador.jsCuota "con gastos y todo": saldo precio + gastos operacionales (parametros_credito) + seguros por tramo + tasa 200 UF. Réplica server-side del simulador (pendiente unificar máxima #1). Usado por Facilito.
cobranzaFullMap (pagos-credito.controller)Mora + gastos de cobranza por cuota al día D. Usado por caja, cobranza, certificados, portal cliente y prepago.
certificados.calcularPrepagoMonto de prepago (cuotas + comisión + interés corriente). Usado por certificado y prepago en caja.
recalcular-mes.js / recalcularPorOpsComisiones e ingresos del negocio; corre solo ante toda vía de cambio; respeta forzados y meses cerrados.
penetracion.js% de penetración de seguros sobre universo elegible por producto.
shared/uf.jsLookup UF por fecha (memoria). Tramo mayor/menor 200 UF se recalcula con UF de fecha_otorgado.
shared/verificacion.jsDocumentos verificables + FES: hash, firmante, QR público, detección de adulteración.
shared/ordenes-pago.jsCorrelativo único global OP- (libro op_correlativos).
contabilidad/src/motor-asientos.jsMotor ÚNICO de asientos automáticos: contabilizar({evento, montos, ref}) — idempotente por origen+ref, nunca lanza (la operación de negocio no se cae por contabilidad). Eventos: PAGO_CAJA, PREPAGO, ODP_PAGADA, CASTIGO, PROVISION_CIERRE, REMUNERACIONES.

Subsistema de IA

shared/anthropic.jsanalizar({codigo, system, prompt, json, max_tokens}) — llama a la API de Anthropic con el modelo asignado a la funcionalidad; gated por ANTHROPIC_API_KEY.
shared/ia.jsRegistro de funcionalidades (registrarFuncionalidad, nacen apagadas), on/off (iaActiva), catálogo ia_modelos (precios) y tracking de tokens/costo por uso (registrarUso).
Funciones livewsp_bot (Haiku, conversación Facilito), informe_crediticio (Sonnet, Reporte Crediticio Automático), liq_sueldo, carga_validador, wsp_plantillas (Sonnet 5, revisor de plantillas HSM), evaluación de consistencia.

Patrón de herramientas del bot: la IA devuelve JSON con intención (simulacion, evaluacion, donde_pagar, derivar, finalizar) y el código ejecuta el cálculo/consulta y redacta el resultado — la IA nunca inventa cifras ni ve informes.

Modo Desarrolloshared/dev-mode.js: redirige correo y WhatsApp a destinos de prueba; cinta DESARROLLO en la UI. Todo canal nuevo debe consultar getDevMode().
Cuenta break-glassadmin@admin.cl (flag usuarios.protegido) — intencional, exenta de políticas.
CronCorreos programados (módulo propio con on/off), sync indicadores 12h, cobranza-mora 1×/día.
Consola SQL/mantenedores/sql-console/ — SELECT/SHOW/DESCRIBE/EXPLAIN, 1 sentencia, 2000 filas, auditada (Solo Dios).

Convenciones de código

Tablas BD: snake_case · Rutas HTTP: kebab-case · Funciones JS: camelCase · Archivos: kebab-case.js
Respuesta uniforme: { success, data, error }
Pool SIEMPRE: require('../../../shared/config/database')
creditos PK = id (id_credito solo como alias en SELECT)
Cambios quirúrgicos: leer antes de tocar, un cambio a la vez, no tocar lo que funciona
Versionado: subir APP_VERSION en cada cambio frontend (badge = confirmación de deploy)
